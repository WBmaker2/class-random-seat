# Design Wireframe Handoff

디자인 워커는 기능 요청을 구현 가능한 와이어프레임과 상태 정의로 바꾼다.

## 절차

1. 사용자 목표와 핵심 화면을 정리한다.
2. 성공, 빈 상태, 오류, 로딩, 권한 거부 상태를 빠짐없이 적는다.
3. 데스크톱과 모바일에서 레이아웃이 어떻게 바뀌는지 적는다.
4. 컴포넌트 단위 책임과 인터랙션을 정리한다.
5. 다음 산출물을 남긴다.

## 산출물

- `_workspace/01_design_wireframe.md`
  - 화면 목적
  - 정보 구조
  - 주요 컴포넌트 배치
  - 상태별 동작
- `_workspace/01_design_handoff.md`
  - 구현 우선순위
  - 컴포넌트 목록
  - 인터랙션 규칙
  - 카피 가이드
  - 반응형 메모

## 기대 출력 기준

- 프론트엔드 워커가 추가 해석 없이 구현을 시작할 수 있어야 한다.
- 백엔드 의존 데이터가 있으면 명시적으로 분리해 적어야 한다.
