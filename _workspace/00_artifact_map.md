# Fullstack Web Delivery Artifact Map

## Phase 1. Wireframe

- `01_design_wireframe.md`: 화면 목적, 사용자 흐름, 상태 전이, 와이어프레임 서술
- `01_design_handoff.md`: 컴포넌트 목록, 인터랙션, 카피, 반응형 지침

## Phase 2. Contract

- `02_api_contract.md`: 요청/응답 shape, 저장 모델, 인증/권한, 실패 시나리오

## Phase 3. Build

- `03_frontend_handoff.md`: 구현 파일, UI 상태, 미해결 의존성
- `03_backend_handoff.md`: 데이터 계층 변경, 환경 의존성, 계약 메모

## Phase 4. QA

- `04_qa_findings.md`: 재현 절차, 실행 명령, 결과, 릴리즈 차단 여부

## Phase 5. Release

- `05_release_notes.md`: 반영 범위, 검증 결과, 배포 URL, 잔여 리스크
